# Worklog

Worklog documents are running, chronological logs of work sessions, kept separate from per-topic implementation documents.

Use this directory when the narrative of what happened, when, and why should survive on its own. Append one entry per session and never rewrite past entries. Name files by topic slug, and add a date or ticket suffix when a topic needs more than one log.

Create worklog documents with `./scripts/scaffold.sh create-doc <workspace-name> worklog <topic-slug>`. Use [../../../templates/documents/worklog.md](../../../templates/documents/worklog.md) manually only when the CLI cannot express the specialized file.
