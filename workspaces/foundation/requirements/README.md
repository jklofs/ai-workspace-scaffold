# Requirements

Confirmed requirements for the scaffold itself.

Use this folder when desired behavior for the repository, workspace lifecycle, templates, scripts, or wiki maintenance rules is clear enough to be treated as product intent.

## Phase Scope

**Standalone:** Yes | No
**Reason:** Requirements may be drafted independently when the scaffold behavior is clear, or after discovery when ambiguity must be resolved first.
**Expected Output:** Approved or draft scaffold requirements with acceptance criteria.
**Next Phase:** None | discovery | context | tech-spec | implementation | review
