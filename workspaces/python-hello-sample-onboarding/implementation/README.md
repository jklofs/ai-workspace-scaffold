# Implementation

Implementation documents track execution after requirements and technical intent are clear enough to build.

Use this directory for task status, changed files, implementation decisions, deviations from the tech spec, verification evidence, and session logs.

Create implementation documents with `./scripts/scaffold.sh create-doc <workspace-name> implementation <topic-slug>`. Use [../../../templates/documents/implementation.md](../../../templates/documents/implementation.md) manually only when the CLI cannot express the specialized file.
