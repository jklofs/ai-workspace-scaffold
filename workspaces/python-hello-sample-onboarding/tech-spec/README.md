# Tech Spec

Tech spec documents translate approved requirements into an implementation design.

Use this directory for current system analysis, proposed approach, architecture, data model changes, API changes, UI changes, task breakdown, testing strategy, rollback planning, risks, and review decisions.

Create tech spec documents with `./scripts/scaffold.sh create-doc <workspace-name> tech-spec <topic-slug>`. Use [../../../templates/documents/tech-spec.md](../../../templates/documents/tech-spec.md) manually only when the CLI cannot express the specialized file.
