# Tech Spec

Technical specifications for scaffold changes.

Use this folder when a foundation requirement needs an implementation plan, such as adding scripts, changing templates, restructuring workspace rules, or updating wiki operations.

## Phase Scope

**Standalone:** Yes | No
**Reason:** Technical planning may be standalone for narrow scaffold changes, or it may follow approved requirements for larger changes.
**Expected Output:** Reviewed technical plan, task breakdown, affected files, and verification strategy.
**Next Phase:** None | discovery | context | requirements | implementation | review

## Suggested Files

- `workspace-template-tech-spec.md`
- `wiki-schema-tech-spec.md`
- `workspace-script-tech-spec.md`

## Use For

- Planning template file additions or changes
- Planning scripts under `scripts/`
- Planning updates to `WIKI-SCHEMA.md`, `AGENT-RULES.md`, or agent entry files
- Mapping scaffold requirements to concrete file changes
