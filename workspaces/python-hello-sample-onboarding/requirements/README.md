# Requirements

Requirements documents convert raw input and discovery into confirmed PRD-style product intent.

Use this directory for goals, users or actors, functional requirements, non-functional requirements, acceptance criteria, scope, dependencies, open questions, traceability, and approval status.

Do not write a tech spec from requirements that still contain unresolved ambiguity unless the assumptions are explicit and approved.

Create requirements documents with `./scripts/scaffold.sh create-doc <workspace-name> requirements <topic-slug>`. Use [../../../templates/documents/requirements.md](../../../templates/documents/requirements.md) manually only when the CLI cannot express the specialized file.
