# Review

Review documents verify that implementation satisfies the approved requirements and tech spec.

Use this directory for reviewed document links, acceptance criteria results, test results, issues found, follow-up work, and final approval decisions.

Create review documents with `./scripts/scaffold.sh create-doc <workspace-name> review <topic-slug>`. Use [../../../templates/documents/review.md](../../../templates/documents/review.md) manually only when the CLI cannot express the specialized file.
