# Workspace README Template

Use this structure when creating a new workspace `README.md`.

## Template

```md
# <Project Name>

Short description of the project and why it exists.

## Scope

- What this workspace covers
- What is intentionally out of scope

## Status

- Current phase
- Main open questions
- Immediate next step

## Planning Boundary

- Which implementation repo(s) this workspace may affect
- What decisions are still planning-only and should not yet be applied to code

## Key Documents

- `requirements/`
- `context/`
- `discovery/`
- `design/`
- `tech-spec/`
- `implementation/`
- `review/`
```

## Notes

- Keep the README short and navigational
- Use it to orient humans and agents quickly
- Put detailed analysis in the phase directories, not in the workspace README
